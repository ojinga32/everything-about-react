# React의 기술적인 특징 정리

## 1. React의 기술적 특징

1. **컴포넌트 기반으로 UI를 표현한다.**  
   → UI를 작은 단위의 컴포넌트로 나누어 재사용성과 유지보수성이 좋아진다.

2. **화면 업데이트 구현이 쉽다 (선언형 프로그래밍).**  
   → “무엇을 보여줄지”만 선언하면 되며, React가 나머지를 처리한다.

3. **화면 업데이트가 빠르게 처리된다 (Virtual DOM).**  
   → 실제 DOM 변경을 최소화하고 필요한 부분만 효율적으로 업데이트한다.


---

## 2. 브라우저의 렌더링 과정

```text
HTML  →  DOM
CSS   →  CSSOM
DOM + CSSOM  →  Render Tree  →  Layout  →  Painting
```

자바스크립트 상태 변화가 발생하면  
이 렌더링 과정이 다시 반복된다.


---

## 3. React가 빠른 이유

브라우저 DOM 조작은 비용이 크기 때문에  
**동시에 발생한 업데이트를 모아서 한 번에 DOM에 반영**하는 것이 성능에 중요하다.

React는 **Virtual DOM**을 사용하여 다음을 수행한다:

- 변경점을 계산(diffing)
- 필요한 부분만 업데이트(reconciliation)
- 실제 DOM 조작을 최소화

→ 그 결과 성능이 크게 향상된다.


---

## 4. JSX란?

JSX(JavaScript XML)는  
자바스크립트 안에서 HTML 태그 형태로 UI 구조를 작성할 수 있게 하는 문법이다.

→ 원래 JS 함수에서 HTML 태그를 리턴할 수 없지만  
   **JSX는 이를 가능하게 해주는 문법적 확장**이다.


---

## 5. JSX 작성 시 주의사항

```text
1. 중괄호 {} 내부에는 자바스크립트 "표현식"만 넣을 수 있다.
2. 숫자, 문자열, 배열 값만 렌더링된다. (객체는 렌더링되지 않는다)
3. 모든 태그는 반드시 닫혀 있어야 한다.
4. 반환되는 최상위 태그는 반드시 하나여야 한다.
```

예시:

```jsx
function App() {
  const name = "React";
  return (
    <>
      <h1>Hello {name}</h1>
      <img src="/logo.png" />
    </>
  );
}
```
